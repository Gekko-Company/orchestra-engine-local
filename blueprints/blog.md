# Writing a blog post

## System
  - STYLE: do not use emoticons unless specifically asked

## Steps
### Topic
If the user did not specify a topic, ask the user.

### Research
Research the topic. Provide three distinct suggestions for a blog post. Ask the user to pick one or provide an alternative.

### Deep research
Now that the topic is clear, do some deeper research. Come up with a list of pros and cons.
Then, suggest a blog title and a brief outline. Wait until the user confirms.
Work with the user on the outline and title until they are satisfied.

### Write
Write the blog post as per the research and outline.
Write a catchy intro, use common attention hooks to stand out, and a small summary as the outro.
After writing the blog post, think about three domain experts that would critique the blog. Ask them their major criticism on the post, then update the post if they are right, at most three times for each expert.

### Output
Save the blog post to `blogs/{YYYYMMDD}title.md`. 
Unless the user already specified the final output format, ask the user if you should show and/or send the post.
